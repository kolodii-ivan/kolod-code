# Kolod Code Portfolio Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a portfolio website at `code.ikolodiy.com` branded "Kolod Code" showcasing web apps built with Claude Code.

**Architecture:** Next.js App Router with static site generation. Single scrollable page with dark terminal hero, light project grid, dark footer. Project data stored in a TypeScript array — no database or CMS.

**Tech Stack:** Next.js 15, TypeScript, Tailwind CSS v4, Vercel deployment

**Spec:** `docs/superpowers/specs/2026-03-24-kolod-code-portfolio-design.md`

### Task 8: Compose Main Page

**Files:**
- Modify: `src/app/page.tsx`

- [ ] **Step 1: Compose all sections in page.tsx**

Replace `src/app/page.tsx` with:
```tsx
import Hero from "@/components/Hero";
import ProjectGrid from "@/components/ProjectGrid";
import Footer from "@/components/Footer";
import { projects } from "@/data/projects";

function JsonLd() {
  const data = {
    "@context": "https://schema.org",
    "@type": "WebSite",
    name: "Kolod Code",
    url: "https://code.ikolodiy.com",
    description:
      "Portfolio of web applications crafted with Claude Code by Ivan Kolodii.",
    mainEntity: {
      "@type": "ItemList",
      itemListElement: projects.map((project, index) => ({
        "@type": "ListItem",
        position: index + 1,
        name: project.name,
        url: project.url,
      })),
    },
  };

  return (
    <script
      type="application/ld+json"
      dangerouslySetInnerHTML={{ __html: JSON.stringify(data) }}
    />
  );
}

export default function Home() {
  return (
    <>
      <JsonLd />
      <main>
        <Hero />
        <ProjectGrid />
      </main>
      <Footer />
    </>
  );
}
```

Note: `dangerouslySetInnerHTML` is safe here — the JSON-LD content is entirely static data from our own `projects.ts`, with no user input.
